
X-Update is a module in the XOOPSCube Legacy ecosystem that functions as an update manager (software update service) for the XCL web application platform. The module allows users to update the core system, install and update modules, themes, and preloads.   
This module is designed exclusively for **site administrators**. Regular users do not interact with it.

## Software Update Service

In essence, **X-update** acts like an "App Store" or "Package Manager" specifically tailored for the XCL web application platform, simplifying the often complex process of managing extensions and keeping the core system up-to-date. The update manager contains three main store components that manage different types of updates.

## Deployment Methods

All three store components use the same deployment methods:

- Direct Method: Direct file deployment
- FTP Method: Deployment via FTP (with optional SSL)
- SSH Method: Deployment via SSH/SFTP

The admin menu in X-Update provides access to all these store components.


## Deployment Workflow

X-Update is a 'Trust module' (D3 module) installed in your `trust_path` directory. This location outside the web root is crucial because the module performs sensitive file operations (downloads, writes) during updates and installs, which reduces the attack surface and enhances overall security.

## Update Process

1. **Checking for Updates:**  
Detecting available updates for the XOOPS Cube core, installed modules, themes, and preloads.

2. **Installing New Add-ons:**  
Browse, downloading, and installing new modules, themes, and preloads from designated repositories (often referred to as a "Store").

3.  **Updating Existing Add-ons:**  
Downloading and applying updates to already installed components.

4. **Managing Add-ons:**  
Providing an interface to view installed components and potentially manage them (though full management might involve other core features).


### Key Features

1. **Store/Repository Connection:**  
X-Update connects to one or more predefined external URLs (configured in its admin settings). These URLs point to repositories that list available XOOPSCube add-ons and their versions.

2. **Metadata Fetching:**  
It fetches metadata from these repositories. This metadata contains information about available add-ons, their descriptions, compatible XOOPSCube versions, current update versions, download links, etc.

3. **Version Comparison:**  
It compares the versions of the installed components (core, modules, themes) on the local XCL site with the latest versions available listed in the fetched metadata.

4. **Update Notification:**  
It presents a list of available updates to the administrator.

5. **Add-on Browser:**  
It typically provides an interface (the "Store") to browse and search for new modules and themes available in the configured repositories.

6.  **Download & Extraction:**  
When an update or new installation is requested, X-Update downloads the necessary package (usually a zip or tar.gz archive) from the repository. It then extracts the contents of the archive.

7. **File Placement:**  
It intelligently places the extracted files into the correct locations within the XCL installation (e.g., module files into `html/modules/` and `xoops_trust_path/modules/`, theme files into `html/themes/`).

8. **Installation/Update Scripts:**  
It executes any installation or update scripts provided within the package. These scripts often handle tasks like:

  * Creating or altering database tables.
  * Updating module version information in the database.
  * Running configuration tasks.

9.  **Status Reporting:**  
It reports the success or failure of the update/installation process to the administrator.


## Configuration Options

X-Update provides various configuration options to control how updates are fetched and deployed:

- FTP, SSH2, or direct file methods for deployments
- SSL verification settings for secure connections
- Custom permissions for files and directories
- Repository URL configurations
- Proxy settings

Deployment options such as SFTP, SSH, and archiving offer different approaches to transferring and deploying software or data. SFTP and SSH provide secure, encrypted connections for file transfer and remote access, while archiving is a method of packing and compressing files for storage and later retrieval


## Notification System

The module includes a notification system that alerts administrators about available updates:

- Admin Notifications: Displays available updates in the admin panel
- Frontend Block: Optional block that can be installed to show updates on the frontend

?> X-Update's components work together to monitor, notify, and deploy updates for different types of add-ons  
from central repositories.


## Technical Implementation

X-Update uses a handler system to manage different aspects of the update process. For example, the `ModuleStoreHandler` is responsible for checking if modules have updates available and managing the installation process.

The notification system alerts administrators about available updates through a sticky notification or on-demand notifications in the admin panel.

For secure connections, `XupdateUtils` provides options to configure SSL verification and proxy settings.


## Troubleshooting Guide

This section addresses common issues encountered with the X-Update module.

### Resolving SSL Verification Errors

Failures during update checks or component downloads may be caused by SSL/TLS certificate verification issues.  
If you encounter such errors, consider the following steps:

1. **Verify Module Preferences**: First, review the X-Update module's configuration settings related to SSL/TLS verification. Ensure these preferences are appropriate for your server environment and repository sources.
2. **Update Server CA Certificate Bundle**: Confirm that the Certificate Authority (CA) bundle on the web server is current. Outdated CA bundles are a frequent cause of verification failures. Refer to your operating system or hosting provider documentation for instructions on updating the CA bundle.
3. **Manually Add Required CA Certificates**: If the update repository uses a certificate issued by a private CA or one whose root/intermediate certificate is not included in your server's standard bundle, manually add the necessary CA certificate(s) to your server's trust store.
4. **Investigate Server-Side Configuration**: Check for server-side factors that could impede outbound HTTPS connections. This includes firewall rules, proxy server configurations, DNS resolution issues, or specific PHP/cURL extension settings related to SSL.
5. **Disable cURL SSL Peer Verification (Temporary Diagnostic / High-Risk Workaround)**:
  - Use with Caution: In some configurations, X-Update settings might allow disabling SSL peer verification for cURL. This can be used as a temporary diagnostic step to isolate the problem or as a last resort in controlled environments.
  - Security Warning: Disabling this verification eliminates protection against man-in-the-middle attacks and is strongly discouraged for production systems. The underlying certificate validation issue should always be addressed for proper security.
6. **Attempt with Alternative Transfer Tool (If Available)**: If X-Update offers configuration options to utilize different download tools (e.g., wget instead of cURL), try selecting the alternative tool to see if it bypasses the issue.

### Key Takeaways

In essence, `xupdate` acts like an "App Store" or "Package Manager", simplifying the often complex process of managing extensions and keeping the core system up-to-date.

- A software update service (SUS) to quickly update web applications and extend them with add-ons
- Download critical updates and security patches as soon as they become available
- A trust module in the XCL package
- The module can be extended to create and manage your own Update Store.

Please note that X-Update is designed for installing/updating components, not for creating them or setting up component stores. Information on creating your own store can be found in the documentation section of the software update service **create you store**
