The architecture consists of:

- XoopsDatabase: Abstract base class defining the interface
- XoopsMySQLDatabase: MySQL implementation of the database interface
- XoopsMySQLDatabaseSafe: Ensures all queries use the secure queryF() method
- XoopsMySQLDatabaseProxy: Read-only mode that only allows SELECT queries
- XoopsDatabaseFactory: Creates the appropriate database connection

## Database Connection Process

To obtain a database connection, applications should use the XoopsDatabaseFactory

```php
$db =& XoopsDatabaseFactory::getDatabaseConnection();
```

This returns a singleton instance of the database handler, ensuring consistent connection management.

### Query Execution

The database abstraction layer provides several methods for executing SQL queries

- query($sql, $limit, $start): Execute a query (implementation varies by handler)
- queryF($sql, $limit, $start): Execute a query and return the result resource
- queryFromFile($file): Execute queries from an SQL file

For working with results:

- fetchArray($result): Fetch a result row as an associative array
- fetchRow($result): Get a result row as an enumerated array
- getRowsNum($result): Get number of rows in result
- getInsertId(): Get ID generated from the previous INSERT operation


### Prepared Statement Emulation

The database layer includes an emulation of prepared statements to help prevent SQL injection:

```php
$db->prepare("SELECT * FROM users WHERE uid = ?");
$db->bind_param("i", $userId);
$result = $db->executeF();
```
Parameters are bound with type information:

- i: Integer
- s: String
- d: Double

### Client Object Handler Features

The Legacy_AbstractClientObjectHandler extends the basic object handler with integrations to other services:

- Tag Management: Associates tags with content objects
- Image Handling: Manages image uploads and associations
- Activity Tracking: Records user activity
- Workflow Integration: Supports content approval workflows
- Map Integration: Associates geographical coordinates with content

When objects are saved or deleted, these operations are handled automatically:

```php
// Saving an object with client handler features
$handler = xoops_gethandler('mymodule_content');
$obj = $handler->create();
$obj->set('title', 'New Content');
$obj->mTag = ['tag1', 'tag2']; // Tags will be saved automatically
$handler->insert($obj);
```

### Text Sanitization Features

1. HTML Escaping: Converts special characters to HTML entities

```php
$text = $textFilter->toShow($text);
```

2. Textarea Processing: Processes textarea content for display

```php
$text = $textFilter->toShowTarea($text, $html, $smiley, $xcode, $image, $br);
```

3. HTML Purification: Cleans HTML content using HTMLPurifier

```php
$text = $textFilter->purifyHtml($html);
```

4. XCode Processing: Handles BBCode-style formatting tags

```php
$text = $textFilter->convertXCode($text, $allowimage);
```

5. UTF-8 Support: Proper handling of UTF-8 character encoding

### Example Integration Flow

When a module saves data using a client object handler:

```php
// In a module:
$handler = xoops_gethandler('mymodule_article');
$article = $handler->create();
$article->set('title', 'New Article');
$article->set('content', 'Article content');
$article->mTag = ['news', 'announcement']; // Tags will be processed
$handler->insert($article);
```

The `Legacy_AbstractClientObjectHandler` detects configuration settings and automatically integrates with other systems:

```php
// Automatic processing in the handler:
if ($this->_isTagClient($conf)) {
    $this->_saveTags($obj, $conf[$this->_mClientConfig['tag']]);
}

if ($this->_isImageClient($conf)) {
    $this->_saveImages($obj);
}

if ($this->_isActivityClient($conf)) {
    $this->_saveActivity($obj);
}
```

### Key Takeaways

XOOPSCube Legacy's data handling system provides a comprehensive framework for:

- Database abstraction: Uniform interface to the database
- Object persistence: OOP approach to data management
- Query building: Secure and flexible query construction
- Text filtering: Input/output sanitization and formatting
- Template integration: Database-driven template system
- Caching mechanism: Performance optimization

These components work together to enable secure, flexible, and efficient data handling throughout the framework. The delegate-based extension system allows modules to customize behavior while maintaining consistent interfaces.

Understanding these data handling components is essential for developing modules for XOOPSCube Legacy or extending the core framework.
